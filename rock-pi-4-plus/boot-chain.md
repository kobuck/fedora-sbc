# Boot Chain — Radxa Rock Pi 4 Plus

**SoC:** Rockchip RK3399 (2× Cortex-A72 + 4× Cortex-A53)
**Arch:** aarch64
**RAM:** 4GB LPDDR4
**Board revision:** v1.73
**OS:** Fedora Linux 43 (Forty Three)
**Kernel:** 6.19.9-200.fc43.aarch64 (Fedora-packaged)
**Status:** ✅ Working

---

## Boot Stages

```
SD card raw offsets
  ├─ sector 64     — idbloader.img (DDR init + U-Boot SPL)
  └─ sector 16384  — u-boot.itb (U-Boot proper + TF-A BL31 FIT image)
       ↓ UEFI environment presented to OS
  systemd-boot (EFI/systemd/systemd-bootaa64.efi)
       ↓ BLS entry loaded
  Linux 6.19.9-200.fc43.aarch64 + dracut initramfs
       ↓
  Fedora 43 aarch64 rootfs (ext4, UUID in root=)
```

---

## Stage Details

### Stage 1 — Boot ROM (RK3399)

RK3399 boot ROM reads the bootloader from raw SD offset — no partition
type GUID required (contrast with JH7110 which uses partition type GUIDs).
The boot ROM loads idbloader from sector 64.

### Stage 2 — idbloader (DDR init + SPL)

`idbloader.img` is a Rockchip-format image combining the DDR initialization
blob and U-Boot SPL. Built from mainline U-Boot with `rock-pi-4-rk3399_defconfig`.

Written to raw SD offset:
```bash
dd if=idbloader.img of=/dev/sdX seek=64 bs=512 conv=notrunc
```

### Stage 3 — U-Boot + TF-A BL31

`u-boot.itb` is a FIT image containing U-Boot proper and the TF-A BL31
(ARM Trusted Firmware) binary for RK3399. SPL verifies hashes of all
components before loading.

Written to raw SD offset:
```bash
dd if=u-boot.itb of=/dev/sdX seek=16384 bs=512 conv=notrunc
```

**Important:** `sgdisk --zap-all` when partitioning will overwrite these
raw offsets. Always reflash U-Boot after partitioning.

**U-Boot config additions required:**
- `CONFIG_EFI_LOADER=y` — presents UEFI environment to OS
- `CONFIG_EFI_RT_VOLATILE_STORE=y` — required for writable efivarfs

### Stage 4 — systemd-boot

U-Boot hands a UEFI environment to systemd-boot. Installed via
`bootctl install` on the live system, which registers the "Linux Boot
Manager" EFI boot entry and places the binary at the canonical path.

systemd-boot reads Boot Loader Specification (BLS) Type #1 entries from
the ESP and loads the kernel, initrd, and kernel command line.

### Stage 5 — Linux 6.19.9 + dracut initramfs

RK3399 aarch64 is fully supported in standard Fedora package repositories.
The kernel is installed via `dnf install kernel` — no custom build required.
dracut generates the initramfs automatically via kernel-install.

Because an initramfs is present, `root=UUID=...` in the kernel command line
is safe — dracut's early userspace resolves UUID references before rootfs mount.

**kernel-install path on Fedora:**
```
/usr/lib/modules/<version>/vmlinuz   ← kernel image
/usr/lib/modules/<version>/initrd    ← generated by dracut
```
BLS entry placed at `/boot/efi/<machine-id>/<version>/`.

## Partition Layout

| Partition | Start | Size | Type | Label | Purpose |
|-----------|-------|------|------|-------|---------|
| (raw) | sector 64 | 186KB | — | — | idbloader (DDR+SPL) |
| (raw) | sector 16384 | 1.3MB | — | — | U-Boot FIT + TF-A BL31 |
| p1 | sector 32768 | 512MB | EF00 | EFI | ESP — systemd-boot + kernel |
| p2 | — | remainder | 8300 | rootfs | Fedora rootfs (ext4) |

Partitions start at sector 32768 (16MB) — safely beyond both raw U-Boot
images. Note that `sgdisk --zap-all` wipes sectors 64 and 16384 — reflash
U-Boot after any repartition operation.

## Build Environment

- **Host:** x86_64 builder VM (ARMbldr), Fedora 43, 16 vCPU / 16GB RAM
- **Cross-compiler:** gcc-aarch64-linux-gnu 15.2.1
- **TF-A:** v2.12.0, `PLAT=rk3399`, `M0_CROSS_COMPILE=arm-linux-gnu-`
  (Fedora does not ship `arm-none-eabi-gcc`; `arm-linux-gnu-gcc` works)
- **U-Boot:** v2026.01, `rock-pi-4-rk3399_defconfig`

## Notes

- **U-Boot board identification:** U-Boot 2026.01 identifies the board as
  "Rock Pi 4A" — the mainline DTS does not yet have a dedicated Rock Pi 4
  Plus entry. Functionally equivalent for boot purposes; the hardware
  difference (onboard eMMC) is separately accessible.

- **onboard eMMC:** Rock Pi 4 Plus has 28.9GB onboard eMMC (not present
  on 4A/4B). Currently unused — available as a future rootfs migration
  target to free the SD slot.

- **PCIe timeout at boot:** `rockchip-pcie f8000000.pcie: PCIe link
  training gen1 timeout` appears in journal — harmless when no PCIe
  device is attached.

- **DMC frequency driver:** `rk3399-dmc-freq: Invalid operating-points`
  — missing OPP table in mainline DTS for the memory controller frequency
  scaling driver. Non-critical; board runs at fixed DDR frequency.
